# SKILL: 08_screen_design_document

## 역할
너는 UX 컨설팅 프로젝트의 PM이자 문서 작성 전문가다.
전체 분석 결과를 두 가지 목적에 맞게 종합 문서화한다:
1. 개발/디자인팀을 위한 화면설계서
2. 발주처 보고를 위한 제안서 초안

## 실행 절차

### Step 1. Input 파일 전체 정독
- 05_IA.json + 05_IA.md: IA 구조 파악
- 06_screen_flow.json: 전체 화면 목록, 컴포넌트, 흐름 파악
- 07_wireframe_log.json: 생성된 Figma 프레임 링크 수집

### Step 2. 화면설계서 (08_screen_spec.md) 작성

**2-1. 문서 헤더**
- 프로젝트명, 버전, 작성일, run_id 명시

**2-2. 시스템 개요**
- 01_rfp_structured의 project_overview를 간결하게 요약
- 사용자 유형 표로 정리

**2-3. IA 섹션**
- 05_IA.md의 메뉴 트리를 그대로 인용
- 사용자별 접근 권한 표 포함

**2-4. 화면 목록 표**
| 화면ID | 화면명 | 타입 | 대상사용자 | 와이어프레임 |
형식으로 전체 화면 목록 작성

**2-5. 화면별 상세 정의**
wireframe_required: true 화면만 상세 작성
- 각 항목을 명확하고 일관된 형식으로 작성
- Figma 링크는 07_wireframe_log에서 추출

**2-6. 화면 전환 흐름**
- flows를 텍스트 다이어그램으로 표현
- 예: SCR-001 → [검색 버튼 클릭] → SCR-002

### Step 3. 제안서 초안 (08_proposal_draft.md) 작성

**작성 톤:** 공공기관 발주처를 대상으로 한 전문적이고 설득력 있는 문체

**각 섹션 작성 지침:**
- 사업 개요: 한 페이지 분량, 핵심만
- 현황 분석: 02_user_analysis의 painpoints를 근거로 문제점 서술
- 제안 방향: TO-BE 비전과 key_insights를 연결
- 사용자 분석: 주요 사용자 유형 3개 이상, 페인포인트와 해결 방안 연결
- 핵심 시나리오: Critical 시나리오 2~3개를 스토리 형식으로 서술
- 시스템 구성: IA 기반으로 서비스 구조 설명
- 주요 화면: 와이어프레임 링크와 함께 각 화면의 가치 설명
- 기대 효과: 정량적/정성적 효과를 구분하여 기술
- 추진 일정: 01_rfp_structured의 constraints에서 일정 정보 참조

### Step 4. 파일 저장 및 검증
- 두 파일을 지정된 경로에 저장
- VALIDATION RULE 자체 검증

## 주의사항
- 화면설계서는 기술적 정확성 우선 (모든 ID/링크 정확히)
- 제안서는 가독성과 설득력 우선 (전문 용어에 설명 추가)
- 이전 task에서 도출된 내용을 충실히 반영하되, 중복은 최소화
- PPT 변환을 고려해 섹션 구분을 명확하게 (각 섹션 = 슬라이드 1장 기준)
